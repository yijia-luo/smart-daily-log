＃ 生成观代日历＋待办工具 Skill

# 目的
本 skill 用于通过 codex 一步生成一个完整的现代日历+待办事项网页工具。该工具提供了多视图的任务管理系统，支持日历选擇、任务管理、每日和月度总结、类型分类、以及定时提醒等功能。

## 核心功能模块

### 1. 日历视图
- 交互式日历网格，支持年份和月份选择
- 点击日期可切换当前选中日期
- 日期徽章显示该日期的任务数量
- 当日任务区域：展示选中日期的所有任务
- 当日总结/备忘区域：记录该日的完成进度、灵感、提醒或反思
- 每日提醒时间设置（⌚️ 按钮），可点击展开时间输入框

### 2. 待办视图
- 新增任务：输入任务名称、选择日期、选择时间、指定任务类型
- 编辑/删除任务：支持点击任务进行编辑或删除
- 完成任务：点击任务左侧复选框标记为完成，完成时触发庆祝反馈（语音表扬 + 烟火动画）
- 排序功能：支持按日期升序/降序、按时间升序/降序、按状态（未完成/已完成优先）排序
- 筛选功能：可按状态（全部/未完成/已完成）和分类（全部或特定类型）筛选
- 统计显示：顶部显示已完成任务数和总任务数

### 3.总结视图
- 月份选择器：用户可选择要查看的月份
- 月度每日总结列表：展示该月每一天的总结内容，可直接编辑并自动保存
- 月度总结文本框：编辑该月的整体总结内容
- 月度总结保存自动持久化

### 4. 类型管理视图
- 创建任务类型：输入类型名称
- 颜色选择：从预设调色板中选择该类型的显示颜色
- 类型列表：显示所有已创建的任务类型及其颜色
- 删除类型：支持删除不需要的任务类型

### 5. 数据存储
所有数据使用 `Localstorage` 持久化、包括：
- 任务列表(包含任务名、日期、时间、类型、完成状态)
- 任务类型及其颜色设置
- 每日总结内容(按日期键值存储)
- 月度总结内容(按月份键值存储)
- 每日提醒时间设置
- 上次提醒状态记录(防止重复提醒)

### 6. 提醒功能
- 用户可在日历视图中设置每日提醒时间
- 当到达设定时间且该日还未编写总结时，页面弹出提醒横幅
- 提醒附带音效（双振荡器合成的有冲击力的警报音）
- 每日仅提醒一次，井记录提醒状态以防重复

## 技术实现细节

### UI 架构
- 使用 HTML5 多视图标签页设计 （日历、待办、总结、类型）
- CSS 现代设计语言：蓝色调色板、圆角、阴影、毛玻璃效果
- 响应式布局，最大宽度 760px，居中显示，支持移动端

### 页面元寨组织
- 顶部导航条：显示应用标题、口号和统计信息
- 视图切换标签页：四个主要功能区域的快速切换
- 各初圏内容面板：独立渲染各功能区域

### 颜色配置
预设10种任务类型颜色，支持用户创建类型时从中选择：
- 蓝色、绿色、橙色、紫色、青色、黄色、粉红色、深紫、深绿、金色

### 日期処理
- 日期格式统一为 YYYY-MM-DD
- 月份格式统一为 YYYY-MM
- 支持跨年选择和任意年份范国 (1997-2097)

### 状态管理与渲染
- 使用全局状态对象维护：当前选中日期、选中任务、编铜中的任务 ID
- DOM 渲染西数按需刷新，确保数据与视因同步
- 事件监听鄉定到视图切换、数据修改等关键操作

### 交互反馈
- 任务完成时：触发语音合成（中文表扬词）和烟火粒子动面
- 提醒时：显示页面级提醒横幅和合成警报音
- 表单校验：创建任务时检查必填字段，给出错误提示

## 文件结构
生成三个核心文件：

1. **HTML文者** (`index.html`)
    - 定义页面结构、表单、列表容器和视图面板
    - 引人 CSS 様式表和 JavaScript 脚本

2. **CSS 様式表**(`styles.css`)
    - 定义 CSS 变量（颜色、阴影、间距）
    - 实现视图切换、表单、日历网格、任务卡片、提醒横幅、烟火动画等样式
    - 支持响应式布局和现代视觉效果
3. **应用逻辑脚本**(`app.js`)
    - DOM 元素选择和全局状态初始化
    - LocalStorage 持久化操作（保存/速取）
    - 核心业务逻辑西数（新增/编辑/删除任务，类型管理）
    - 日期处理和日历渲染
    - 各视图的 DOM 渲染函数
    - 事件监听和交互处理
    - 提醒检查和音效搔放

# 使用指南
1. 在需要的位置创建工作目录
2. 通过本 skill 调用 Codex 生成三个文件
3. 在浏览器中打开生成的 HTML 文件即可使用
4. 所有数据自动保存到浏览器 LocaLStorage，刷新页面后数据保留

# 生成规则
- 生成 HTML、 CSS、Javascript 三个文件
- 所有代码为原生浏览器实现，不依赖任何外部库或后端服务
- 代码应包含完整的功能实现，无占位符或待完成部分
- 所有数据操作使用 localStorage，支持离线使用
